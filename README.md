# Описание

Приложение реализует функционал небольшого веб-сервиса, который, используя внешние веб-сервисы, по запросу возвращает случайную фотографию собаки, кота или лисы, предварительно обработав фотографию на сервере.

# Руководство по запуску приложения

Запуск приложения производится путём запуска flask веб-сервера, после создания виртуального окружения и установки нужных пакетов.

### Сценарий запуска для Linux:
Создаём и активируем виртуальное окружение:
```bash
cd sources
python3 -m venv test_env
source test_env/bin/activate
```
Скачиваем и устанавливаем нужные пакеты:
```bash
pip install -e .
```
Запускаем flask веб-сервер
```bash
python3 -m flask run
```

### Сценарий запуска для Windows
Создаём и активируем виртуальное окружение:
```bash
cd sources
python -m venv test_env
test_env\Scripts\activate
```
Скачиваем и устанавливаем нужные пакеты:
```bash
pip install -e .
```
Запускаем flask веб-сервер
```bash
python -m flask run
```

# Руководство пользователя

Веб-сервис отвечает на запросы вида **'/animal/[cat|dog|fox]'** случайной фотографией с котом, собакой, или лисой. К фотографиям применяется фильтр блюра (размытие). Фотография будет скачана в папку **images** и соответствующая запись будет сделана в БД.
Полные варианты запросов для получения фотографий:
* /animal/cat
* /animal/dog
* /animal/fox

Также веб-сервис может предоставить пользователю полную историю запросов фотографий животных. 
История будет представлена в виде списка записей БД с таким синтаксисом: \<\<id\>:\<animal_type\> \<uuid\> - \<created_on\>\>.
Запрос истории:
* /history

Получить ранее загруженную фотографию можно указав её uuid (имя файла без расширения) в данном запросе:
* /history/static/\<uuid\>

_Note: узнать uuid конкретной фотографии можно из истории запросов ('/history')_
